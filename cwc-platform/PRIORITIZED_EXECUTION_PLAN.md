# CWC Platform Prioritized Execution Plan

This plan converts the platform checklist into an execution sequence designed to get Wendy using the platform first, then expand CWC into a stronger all-in-one product and future white-label SaaS.

## Execution Principle

Do not build everything at once.

Sequence the work in this order:
- make Wendy successful in daily use
- replace the most painful tools in the current stack
- build the platform differentiators
- then productize for white-label SaaS

## Track 1: Must Have For Wendy

These items should be treated as the immediate operating system for CWC.

### Goal
Wendy can use the platform for real work every day without needing to bounce between multiple tools for core workflows.

### Highest Priority
- [ ] Stable login and session handling
- [ ] Pilot environment Wendy can trust
- [ ] Active client data available in the system
- [ ] Booking types configured for real offers
- [ ] Contracts ready for real use
- [ ] Invoicing/payment settings ready for real use
- [ ] Dashboard and navigation clear enough for daily use
- [ ] Notes/tasks/follow-up workflow reliable

### Wendy Day-1 Workflow Target
- [ ] Find client
- [ ] Update client record
- [ ] Send contract
- [ ] Send invoice
- [ ] View bookings/calendar
- [ ] Capture notes and next steps

### Pilot Guardrails
- [ ] Define which workflows stay in old tools for week 1
- [ ] Define fallback if one module breaks
- [ ] Back up pilot data before heavy use
- [ ] Create short quick-start instructions for Wendy
- [ ] Schedule daily issue review during pilot week

## Track 2: Replace The Scheduling Stack

This is one of the biggest wins because Wendy is currently dealing with Calendly, Motion, and multiple calendars.

### Goal
One system for booking and one system for availability truth.

### Immediate Build Sequence
- [ ] Audit current scheduling setup: Calendly, Motion, Google, Microsoft, personal/business calendars
- [ ] Document the exact scheduling pain points
- [ ] Support multiple calendar connections per user
- [ ] Aggregate busy time across all connected calendars
- [ ] Create one unified availability engine
- [ ] Prevent double-booking across all connected calendars
- [ ] Allow one primary calendar for write-back

### Integration Priorities
- [ ] Google personal calendar
- [ ] Google Workspace calendar
- [ ] Microsoft personal calendar
- [ ] Microsoft 365 business calendar
- [ ] Motion compatibility strategy
- [ ] ICS fallback/import support

### Booking Link Priorities
- [ ] Brandable booking pages
- [ ] Logo/colors/typography controls
- [ ] Better messaging on booking pages
- [ ] Better confirmation UX
- [ ] Email branding for booking confirmations
- [ ] Multi-coach routing support

### Outcome
Wendy can stop relying on disconnected scheduling tools and present one branded booking experience.

## Track 3: Coaching Session Intelligence

This is a high-value differentiator because it connects delivery, automation, client experience, and certification tracking.

### Goal
Turn completed coaching sessions into structured records automatically.

### Immediate Priorities
- [ ] Zoom ingestion
- [ ] Fathom transcript/note ingestion
- [ ] Match sessions to bookings
- [ ] Match sessions to clients
- [ ] Create internal session record automatically

### Client Experience Layer
- [ ] Publish approved session notes to the client portal
- [ ] Publish approved transcripts to the client portal
- [ ] Publish approved recordings to the client portal
- [ ] Generate homework/action items automatically
- [ ] Generate post-session recap automatically

### ICF Automation Layer
- [ ] Auto-log ICF hours from completed sessions
- [ ] Session type classification
- [ ] Review/override workflow
- [ ] ICF dashboard updates from meeting data

### Controls
- [ ] Coach-only notes
- [ ] Client-visible summary
- [ ] Client-visible transcript
- [ ] Client-visible recording
- [ ] Approval workflow before auto-sharing

### Back Burner
- [ ] Google Meet ingestion
- [ ] Microsoft Teams ingestion
- [ ] Other meeting platforms

## Track 4: Finance, Bookkeeping, and Tax Readiness

This is the operations layer that moves the product from “coaching platform” to “business operating system.”

### Goal
Replace fragmented financial admin and create CPA-ready business records.

### Finance MVP
- [ ] Unified ledger for invoices, payments, refunds, expenses, contractor payouts
- [ ] Chart-of-accounts-lite for coaches
- [ ] Expense entry workflow
- [ ] Contractor payment tracking
- [ ] Mileage and reimbursement support
- [ ] Cash flow dashboard
- [ ] Monthly P&L

### Tax Readiness
- [ ] Tax-aware categorization defaults
- [ ] Quarterly tax estimate view
- [ ] Year-end contractor totals
- [ ] CPA export package
- [ ] Audit log for financial changes

### Bookkeeping Workflow
- [ ] Receipt attachment support
- [ ] Bank import support
- [ ] Reconciliation workflow
- [ ] Matched/unmatched transaction states
- [ ] Close-the-month checklist

### Why This Matters
This is one of the clearest reasons a coach would switch from a pile of tools to one operating system.

## Track 5: Community and Content

This is the Kajabi/Circle/Skool replacement direction, but it should follow the operational core, not precede it.

### Goal
Keep client learning, community, and engagement inside the same platform as business operations.

### Priorities
- [ ] Community space inside the platform
- [ ] Cohort/program groups
- [ ] Content library/course delivery
- [ ] Events/live-session access
- [ ] Discussion feed
- [ ] Access control tied to program/payment status

### Integration Priorities
- [ ] Tie community access to CRM/contact/program records
- [ ] Tie content access to billing/subscription status
- [ ] Shared reporting across delivery and revenue

### Positioning Rule
Do not try to beat Kajabi everywhere. Build enough to eliminate tool sprawl for service-based coaching businesses.

## Track 6: White-Label Productization

This comes after CWC proves the workflows in live use.

### Goal
Turn the CWC operating system into a white-label platform other coaches can run their businesses on.

### Architecture Priorities
- [ ] Tenant/workspace model
- [ ] Data isolation
- [ ] Permissions per tenant
- [ ] Tenant settings
- [ ] Tenant branding
- [ ] Tenant domains

### SaaS Priorities
- [ ] Tenant billing/plans
- [ ] Self-serve onboarding
- [ ] Trial/demo flow
- [ ] Feature gating
- [ ] Internal support/admin tooling

## Track 7: Marketing Funnels and Email Growth

This is the Mailchimp/Kajabi-style capability, but it should come after the operational core is trusted.

### Goal
Keep list growth, nurture, campaigns, and offer conversion inside the same platform as CRM, bookings, payments, and delivery.

### Priorities
- [ ] Audience lists and CRM-driven segmentation
- [ ] Tags based on lifecycle stage, offer, and engagement
- [ ] Opt-in forms and lead capture flows
- [ ] Landing pages for lead magnets, webinars, and offers
- [ ] Broadcast campaign sending
- [ ] Automated nurture and follow-up sequences
- [ ] Funnel analytics from opt-in to booking to purchase
- [ ] Triggered campaigns from booking, contract, invoice, and portal events

### Positioning Rule
- [ ] Do not try to out-Mailchimp Mailchimp on generic email marketing
- [ ] Do not try to out-Kajabi Kajabi on creator marketing breadth first
- [ ] Build the version that is tightly integrated with coaching-business operations

3. Coaching session intelligence and client delivery

## Simple Rule For Prioritization

If a feature does not help Wendy use the platform more this month, replace a painful tool, or strengthen a major future differentiator, it should not be in the current build wave.
